- 批次: `Abby61000_2025-11-05`
- 正确题库: `R1-27-D3.json` (中翻英)
- 学生样本: Benjamin, Jeffery, Dana

## 测试场景

### 场景1: 正确匹配 ✓
**题库**: R1-27-D3.json (中→英)
```json
{
  "question": "小孩",
  "answer": "kid"
}
```

**ASR转写**:
```
字表二十七英翻中。kid kid 呃小孩。小孩。pupil 小学生，瞳孔...
```

**预期输出**:
```json
{
  "status": "PASS",
  "issue_type": null
}
```

**分析**:
- ✓ 翻译方向正确: question(中文) → answer(英文)
- ✓ ASR 出现 "kid 小孩" (英在前，中在后)
- ✓ 词汇匹配度高 (kid, pupil, baby, twin, leader, friend, classmate 均出现)

---

### 场景2: 翻译方向错误 ✗
**题库**: R1-27-D4.json (英→中)
```json
{
  "question": "kid",
  "answer": "小孩"
}
```

**ASR转写**:
```
字表二十七英翻中。kid kid 呃小孩。小孩。pupil 小学生，瞳孔...
```

**预期输出**:
```json
{
  "status": "FAIL",
  "issue_type": "WRONG_QUESTIONBANK"
}
```

**分析**:
- ✗ 题库方向是 英→中 (question=英文, answer=中文)
- ✗ 但ASR显示老师念的是"英翻中"，实际操作是"kid (读题) → 小孩 (答案)"
- ✗ 这与题库定义的方向相反

## 检测逻辑总结

| 检查项 | 正确题库 | 错误题库D4 | 错误题库D1 | 音频异常 |
|--------|----------|-----------|-----------|---------|
| 题库方向 | 中→英 ✓ | 英→中 ✗ | - | - |
| ASR模式 | "英 中" ✓ | "英 中" ⚠️ | - | - |
| 词汇匹配 | >80% ✓ | >80% ⚠️ | <50% ✗ | - |
| 转写长度 | 充足 ✓ | 充足 ✓ | 充足 ✓ | 过短 ✗ |
| 判定结果 | PASS | FAIL (方向错) | FAIL (词汇错) | FAIL (音频错) |

## 关键检测点

1. **翻译方向识别** (最关键)
   - 分析题库: question/answer 的语言类型
   - 对比ASR: 词汇出现的顺序
   - D3(中→英): ASR应为 "英 中"
   - D4(英→中): ASR若为 "英 中" 则方向相反

2. **词汇匹配度**
   - 统计题库所有词汇
   - 检查转写中出现比例
   - 阈值: >50% 考虑通过, <50% 标记错误

3. **音频完整性**
   - 长度检查: 至少 题库数×5 字符
   - 结构检查: 有完整的"问题-答案"对
   - 声音检查: 包含老师和学生声音
