## 核心发现

### 1. 判断逻辑正确
Gatekeeper 实际上在正确工作：
- **明确不匹配** → FAIL ✓
- **明确匹配** → PASS ✓
- **混乱不确定** → PASS (保守策略) ✓

### 2. 关键理解

**题库方向的含义**:
- D3/D5 (中→英): question=中文, answer=英文
  - 老师念题: 先念中文问题 → 学生回答 → 后念英文答案
  - 正确ASR: "中文词 ... 英文词"
- D4 (英→中): question=英文, answer=中文
  - 老师念题: 先念英文问题 → 学生回答 → 后念中文答案
  - 正确ASR: "英文词 ... 中文词"

**检测方法**:
- 看 ASR 中**先出现的词**是中文还是英文
- 与题库的 question 语言进行对比

## Prompt 改进方向

### 当前 Prompt 的问题
1. 没有明确解释"老师念题流程"
2. 没有明确"ASR 中先出现的词"的意义
3. 示例不够清晰

### 改进建议

#### 1. 明确念题流程
```
老师念题流程:
1. 念 question (问题)
2. 停顿，等学生回答
3. 念 answer (答案)

因此 ASR 中:
- 先出现的词 = question 的语言
- 后出现的词 = answer 的语言
```

#### 2. 明确检测规则
```
检测步骤:
1. 从题库判断方向:
   - question=中文, answer=英文 → 中翻英
   - question=英文, answer=中文 → 英翻中

2. 从 ASR 提取实际模式:
   - 先出现英文词，后出现中文词 → 英翻中
   - 先出现中文词，后出现英文词 → 中翻英

3. 对比两者是否一致:
   - 不一致 → FAIL
   - 一致 → PASS
   - 无法判断 → PASS (保守)
```

#### 3. 增强示例
```
正确示例:
- 题库: question="小孩" answer="kid" (中翻英)
- ASR: "小孩 ... kid" (先中文后英文)
- 结果: PASS ✓

错误示例:
- 题库: question="小孩" answer="kid" (中翻英)
- ASR: "kid ... 小孩" (先英文后中文)
- 结果: FAIL - WRONG_QUESTIONBANK
```

## 结论

Gatekeeper 当前表现良好，判断逻辑正确。需要改进的是 **prompt 的清晰度**，使其：
1. 更明确地解释判断逻辑
2. 提供更清晰的示例
3. 减少模型误判的可能性
