## 功能描述
在 Gemini API 调用前打印完整的 System Prompt 和 User Prompt，方便调试和审查发送给 LLM 的完整提示词内容。

## 修改内容

### 修改函数：`call_gemini_api()`

在函数的第一次尝试（`attempt == 0`）时，添加了以下打印逻辑：

1. **System Prompt 打印**
   - 使用分隔线（80个等号）包围
   - 标题：`🔍 SYSTEM PROMPT (完整内容):`
   - 如果存在 `system_instruction`，打印完整内容
   - 如果不存在，显示 `(无 system instruction)`

2. **User Prompt 打印**
   - 使用分隔线（80个等号）包围
   - 标题：`🔍 USER PROMPT (完整内容):`
   - 打印完整的 user prompt 内容

### 代码位置
`Gemini_annotation.py` 第 114-130 行

## 特性
1. ✅ 只在第一次尝试时打印（避免重试时重复打印）
2. ✅ 打印完整的 system prompt 和 user prompt
3. ✅ 使用清晰的视觉分隔符
4. ✅ 不影响原有的 API 调用逻辑
5. ✅ 不影响日志文件保存功能（`4_llm_prompt_log.txt`）
