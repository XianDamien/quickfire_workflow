## Requirements
### Requirement: ASR 转写结果可观测性

系统 SHALL 在完成音频转写后，将 ASR 原始转写结果输出到控制台。

#### Scenario: 用户查看 ASR 转写输出

- **WHEN** 工作流成功调用 `transcribe_audio()` 完成音频转写
- **THEN** 系统在控制台输出 ASR 转写原始结果，格式如下：
  ```
  ============================================================
  📄 ASR 转写原始结果
  ============================================================
  [ASR 原始文本内容]
  ```
- **AND** 输出在"✅ 音频转写完成"提示之后立即显示
- **AND** ASR 结果的完整内容都应可见（不截断）

#### Scenario: ASR 转写失败时无输出

- **WHEN** 音频转写发生错误并被捕获
- **THEN** 系统显示错误信息，跳过 ASR 结果输出
- **AND** 工作流中止于错误阶段

### Requirement: 题库摘要可观测性

系统 SHALL 在加载题库后，输出题库的摘要信息到控制台。

#### Scenario: 用户查看题库摘要

- **WHEN** 工作流成功调用 `load_qb()` 加载题库数据
- **THEN** 系统在控制台输出题库摘要，包括：
  1. 题库条目总数
  2. CSV 字段列表

  格式如下：
  ```
  ============================================================
  📚 题库摘要
  ============================================================
  题库条目数: [数字]
  字段: [字段列表，例：index,question,expected_answer]
  ```
- **AND** 输出在"✅ 题库加载完成"提示之后立即显示
- **AND** 条目数应准确反映 CSV 行数（去除 header）

#### Scenario: 题库加载失败时无输出

- **WHEN** 题库加载发生错误
- **THEN** 系统显示错误信息，跳过题库摘要输出
- **AND** 工作流中止于错误阶段

### Requirement: AI 评测提示词结构可观测性

系统 SHALL 在调用 AI 评测模型之前，输出完整的提示词结构（三层 prompt）到控制台。

#### Scenario: 用户查看提示词结构

- **WHEN** 工作流准备调用 `evaluate_pronunciation()` 进行 AI 评测
- **THEN** 系统在控制台输出提示词结构，包括三层：
  1. **Layer 1**: System Prompt（系统角色定义）
  2. **Layer 2**: Question Bank Prompt（题库上下文）
  3. **Layer 3**: ASR Data Prompt（ASR 识别结果）

  格式如下：
  ```
  ============================================================
  💬 AI 评测提示词结构
  ============================================================

  [Layer 1] System Prompt (系统角色定义)
  ----------------------------------------
  [前 300-500 字内容] ...

  [Layer 2] Question Bank Prompt (题库上下文)
  ----------------------------------------
  [前 300-500 字内容] ...

  [Layer 3] ASR Data Prompt (ASR 识别结果)
  ----------------------------------------
  [前 300-500 字内容] ...
  ```

- **AND** 如果任何层的内容超过 500 字，应在末尾添加"..."表示省略
- **AND** 输出在第 4 步开始前显示（"✓ 第4步：执行发音评测..."之后）
- **AND** 用户可以通过此输出验证提示词是否正确构成

#### Scenario: 帮助用户调试提示词

- **WHEN** 用户发现 AI 评测结果异常
- **THEN** 用户可以通过查看提示词结构输出，检查：
  - System Prompt 是否定义了正确的角色和规则
  - Question Bank Prompt 是否包含正确的题库数据
  - ASR Data Prompt 是否正确转换了转写结果
- **AND** 用户可以基于此信息进行快速诊断

### Requirement: AI 评测结果可观测性

系统 SHALL 在完成 AI 评测后，将最终评测结果以格式化的 JSON 输出到控制台。

#### Scenario: 用户查看评测结果

- **WHEN** 工作流成功调用 `evaluate_pronunciation()` 完成 AI 评测
- **THEN** 系统在控制台输出评测结果 JSON，格式如下：
  ```
  ============================================================
  📊 AI 评测结果 (JSON)
  ============================================================
  {
    "final_grade_suggestion": "A",
    "mistake_count": {
      "hard_errors": 0,
      "soft_errors": 2
    },
    "annotations": [...]
  }
  ```

- **AND** JSON 应使用缩进（indent=2）以便阅读
- **AND** 中文字符应正确显示（不转义为 Unicode）
- **AND** 输出在"✅ 评测完成"提示之后立即显示

#### Scenario: JSON 解析失败时的降级输出

- **WHEN** AI 返回的结果无法解析为 JSON
- **THEN** 系统原样输出返回的文本内容
- **AND** 不报错，工作流继续进行

#### Scenario: 用户数据验证

- **WHEN** 用户需要验证评测结果的准确性
- **THEN** 用户可以直接在控制台查看完整的 JSON 结果
- **AND** 用户可以复制结果进行进一步分析

---

### Requirement: 可观测性输出的清晰性

系统 SHALL 确保所有可观测性输出格式清晰、易于识别。

#### Scenario: 视觉分隔

- **WHEN** 工作流输出多个可观测性片段
- **THEN** 每个片段应用分隔符（`"=" * 60`）分隔
- **AND** 每个片段应有清晰的标签（如"📄 ASR 转写原始结果"）
- **AND** 不同片段之间应有空行分隔，避免混乱

#### Scenario: 进度清晰

- **WHEN** 用户执行工作流
- **THEN** 用户应能清晰看到：
  1. 当前执行到哪一步（✓ 第X步）
  2. 该步是否成功（✅ 完成或 ❌ 失败）
  3. 该步的 AI 处理结果（新增的可观测性输出）
  4. 整个工作流的进度
