# 任务分解: 工作流可观测性增强

## 核心任务链

### Task 1: 增加 ASR 转写结果输出
**状态**: - [x] **已完成**

**目标**: 在完成音频转写后，将 ASR 原始文本输出到控制台

**实现步骤**:
1. 在 `run_workflow()` 的第2步完成后，添加输出段落
2. 格式化输出：使用分隔符和标签
3. 预期输出示例:
```
============================================================
📄 ASR 转写原始结果
============================================================
spk0: 数字
spk1: number
...
```

**验证方式**:
- ✅ 执行工作流，确认 ASR 结果在控制台显示
- ✅ 检查输出格式清晰可读

**依赖**: 无

**预计工时**: 10 分钟 (实际: ~5 分钟)

---

### Task 2: 增加题库摘要输出
**状态**: - [x] **已完成**

**目标**: 在加载题库后，输出题库摘要信息

**实现步骤**:
1. 在 `run_workflow()` 的第3步完成后，添加输出段落
2. 解析 CSV 数据，提取：题目数量、字段列表
3. 格式化输出

**预期输出示例**:
```
============================================================
📚 题库摘要
============================================================
题库条目数: 65
字段: index,question,expected_answer
```

**验证方式**:
- ✅ 执行工作流，确认题库摘要信息正确显示
- ✅ 验证条目数与实际 CSV 行数匹配

**依赖**: Task 1（顺序性，但无数据依赖）

**预计工时**: 10 分钟 (实际: ~5 分钟)

---

### Task 3: 增加 AI 评测提示词输出
**状态**: - [x] **已完成**

**目标**: 在调用评测 AI 前，输出三层提示词结构

**实现步骤**:
1. 在 `run_workflow()` 的第4步前，添加提示词输出段落
2. 显示三层结构：
   - System Prompt (系统角色定义)
   - Question Bank Prompt (题库上下文)
   - ASR Data Prompt (ASR 识别结果)
3. 由于内容较长，可截断显示（前 500 字）
4. 提供省略号指示有更多内容

**预期输出示例**:
```
============================================================
💬 AI 评测提示词结构
============================================================

[Layer 1] System Prompt (系统角色定义)
----------------------------------------
你是一个专业的AI语言教师助教。你的任务是基于学生提交的录音、对应的题库...

[Layer 2] Question Bank Prompt (题库上下文)
----------------------------------------
本次作业的题库，老师给的标准问题和答案（以csv形式给出）
index,question,expected_answer
1,数字,number
...

[Layer 3] ASR Data Prompt (ASR 识别结果)
----------------------------------------
以下是本音频的ASR数据，包括时间戳和说话人识别数据:
spk0: 数字
spk1: number
...
```

**验证方式**:
- ✅ 执行工作流，确认三层提示词按顺序显示
- ✅ 检查截断逻辑是否正确
- ✅ 验证提示词内容正确性

**依赖**: Task 2（顺序性，但无数据依赖）

**预计工时**: 15 分钟 (实际: ~10 分钟)

---

### Task 4: 增加 AI 评测结果输出
**状态**: - [x] **已完成**

**目标**: 在完成评测后，输出最终 JSON 结果

**实现步骤**:
1. 在 `run_workflow()` 的第4步完成后，添加输出段落
2. 尝试解析 JSON，如可解析则美化输出（`json.dumps(..., indent=2)`)
3. 如解析失败，原样输出
4. 格式化输出

**预期输出示例**:
```
============================================================
📊 AI 评测结果 (JSON)
============================================================
{
  "final_grade_suggestion": "A",
  "mistake_count": {
    "hard_errors": 0,
    "soft_errors": 2
  },
  "annotations": [
    {
      "card_index": 0,
      "question": "数字",
      "expected_answer": "number",
      ...
    }
  ]
}
```

**验证方式**:
- ✅ 执行工作流，确认评测结果 JSON 正确显示
- ✅ 测试 JSON 解析失败情况的降级输出
- ✅ 检查缩进和格式化是否正确

**依赖**: Task 3（顺序性，但无数据依赖）

**预计工时**: 10 分钟 (实际: ~5 分钟)

---

### Task 5: 测试完整工作流
**状态**: - [x] **已完成**

**目标**: 验证所有可观测性输出正常运行

**实现步骤**:
1. 准备测试用音频和题库文件
2. 执行完整工作流
3. 检查所有5个输出段落是否依次显示
4. 验证输出格式、内容准确性
5. 检查没有引入错误或异常

**验证方式**:
- ✅ 端到端测试：工作流执行成功，所有步骤输出完整
- ✅ 边界测试：异常情况（文件不存在等）的处理是否正常
- ✅ 性能验证：新增输出是否显著影响性能
- ✅ 语法验证：通过 `python3 -m py_compile` 检查

**预计工时**: 15 分钟 (实际: ~10 分钟)

**依赖**: Task 1 - Task 4 全部完成

## 可选任务（第二阶段）

### Task 6: 添加详细程度控制（可选）
**目标**: 支持 `--verbose` 和 `--quiet` 标志

**实现步骤**:
1. 修改 `argparse` 配置，添加 `--verbose` 和 `--quiet` 标志
2. 在关键输出点添加条件判断
3. 定义输出级别：
   - DEFAULT: 输出所有中间结果（当前推荐配置）
   - VERBOSE: 输出完整 system_prompt（当前截断）
   - QUIET: 仅输出最终结果

**预计工时**: 20 分钟

**依赖**: Task 5 完成后

---

## 工作流总结

```
Task 1 (ASR 输出)
    ↓
Task 2 (题库输出)
    ↓
Task 3 (提示词输出)
    ↓
Task 4 (评测结果输出)
    ↓
Task 5 (端到端测试)
    ↓
[可选] Task 6 (详细程度控制)
```

**总工时**: ~60 分钟 (核心任务) + ~20 分钟 (可选任务)
