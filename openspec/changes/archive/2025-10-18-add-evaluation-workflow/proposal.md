# 评测工作流集成提案

## Why

目前 `scripts/` 中存在两个独立的脚本：
- `captioner_qwen3.py` - 音频转写工具（必须手动运行）
- `qwen3.py` - 评测引擎（需要提前准备 ASR 转写结果）

这种手动的两步流程效率低下，用户必须：
1. 手动运行音频转写脚本
2. 等待转写完成
3. 手动运行评测脚本

用户需要一个统一的命令行工具，能够：
- 从命令行输入要处理的音频文件路径
- 从命令行指定题库文件路径
- 自动完成"音频转写 → 评测评分"的端到端工作流
- 输出结构化的评测报告（JSON 格式）

## What Changes

- **新增功能**: 创建 `workflow.py` 统一工作流入口
  - 接收命令行参数：`--audio-path`（音频文件）、`--qb-path`（题库路径）
  - 自动调用 `captioner_qwen3.py` 执行音频转写
  - 将转写结果流转到 `qwen3.py` 执行评测
  - 输出最终的 JSON 评测报告

- **重构现有脚本**:
  - 将 `qwen3.py` 中的核心逻辑（加载数据、调用 API、格式化输出）提取为可重用函数
  - 将 `captioner_qwen3.py` 改造为可接收 API 参数并返回结构化结果的模块
  - 保留原脚本的独立可用性（向后兼容）

- **提升用户体验**:
  - 统一的命令行接口
  - 清晰的进度提示和错误处理
  - 支持 system_prompt 硬编码配置

## Impact

- **影响的规范**: 新增 `evaluation-workflow` 能力
- **影响的代码**:
  - `scripts/qwen3.py` - 提取核心函数
  - `scripts/captioner_qwen3.py` - 改造为模块化函数
  - `scripts/workflow.py` - 新建工作流主程序
  - `scripts/__init__.py` - 新建模块包 (可选)
