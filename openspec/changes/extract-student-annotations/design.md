# 学生回答提取功能设计文档

## 系统架构

### 组件关系图
```
教师转录模板 (R3-14-D4_transcription.txt)
         ↓
    题库数据 (R3-14-D4.json)
         ↓
    学生 ASR 结果 (2_qwen_asr.json)
         ↓
    Gemini LLM 处理 (gemini_client.py)
         ↓
    学生回答标注 (4_llm_annotation.json)
```

### 数据流设计

#### 输入数据接口
1. **教师转录模板解析**
   - 格式: 纯文本文件
   - 内容: 【问题】-【停顿】-【答案】模式
   - 用途: 提供问题和答案的顺序参考

2. **题库数据结构**
   - 格式: JSON 数组
   - 字段: `id`, `问题`, `答案`
   - 用途: 提供标准答案和问题索引

3. **学生 ASR 数据**
   - 格式: JSON (DashScope API 响应)
   - 来源: `archive/Zoe51530-9.8/Phoebe/2_qwen_asr.json`
   - 内容: 文本转录结果
   - 用途: 提供学生实际回答内容

#### 输出数据结构
```json
[
  {
    "card_index": 1,           // 题库中的问题索引
    "问题": "simple",          // 题目词汇
    "学生回答": "简单的",       // 提取的学生回答或"未作答"
    "答案": "简单的，简易的；朴素的，简朴的"  // 标准答案
  }
]
```

## 核心算法设计

### 学生回答提取算法

#### 步骤 1: 数据预处理
1. 从 ASR JSON 中提取纯文本内容
2. 清理文本格式，移除不必要的标记
3. 标准化文本编码和格式

#### 步骤 2: 位置匹配
1. 在学生 ASR 文本中识别"问题"关键词位置
2. 在学生 ASR 文本中识别"答案"关键词位置
3. 建立问题和答案的对应关系

#### 步骤 3: 回答内容提取
1. 提取问题和答案之间的文本内容
2. 清理提取的文本（移除停顿词、填充词等）
3. 判断回答的有效性

#### 步骤 4: 结构化输出
1. 匹配题库中的问题索引
2. 构建符合格式的 JSON 对象
3. 处理特殊情况（未作答、识别失败等）

### LLM 提示词集成

#### 系统提示词复用
- 基础角色定义: 数据处理和文本分析专家
- 任务描述: 根据题库、教师转录和学生转录提取回答
- 输出要求: 严格 JSON 格式，包含必需字段

#### 动态内容注入
1. 题库 JSON 数据
2. 教师转录文本
3. 学生 ASR 转写文本
4. 处理指令和格式要求

## 错误处理策略

### 数据缺失处理
- **教师转录文件缺失**: 使用题库顺序作为默认顺序
- **学生 ASR 文件缺失**: 输出空结果或标记所有问题为"未作答"
- **题库文件缺失**: 无法处理，返回错误信息

### 内容匹配失败处理
- **问题匹配失败**: 使用模糊匹配或顺序推断
- **答案匹配失败**: 标记为"未作答"
- **文本提取为空**: 标记为"未作答"

### API 调用错误处理
- **Gemini API 限流**: 复用现有的重试机制
- **JSON 解析失败**: 尝试修复格式或返回错误信息
- **网络连接失败**: 使用备用模型或延迟重试

## 性能优化设计

### 缓存策略
- 题库数据缓存: 避免重复加载
- 教师转录缓存: 复用已解析的结构
- 处理结果缓存: 支持增量处理

### 批量处理设计
- 支持单个学生文件处理
- 预留批量处理接口
- 异步处理可能性
