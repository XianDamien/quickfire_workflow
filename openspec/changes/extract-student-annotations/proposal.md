# 学生回答提取功能提案

## 概述

基于 `prompts/annotation.txt` 提示词模板，改造 `scripts/gemini_client.py` 脚本，实现从学生录音的 ASR 转写结果中自动提取每个问题的学生回答内容。

## 需求描述

### 输入数据
- **教师转录模板**: `archive/Zoe51530-9.8/_shared_context/R3-14-D4_transcription.txt`
- **题库数据**: `archive/Zoe51530-9.8/_shared_context/R3-14-D4.json`
- **学生 ASR 结果**: `archive/Zoe51530-9.8/Phoebe/2_qwen_asr.json`

### 输出数据
- **学生回答标注**: `archive/Zoe51530-9.8/Phoebe/4_llm_annotation.json`

### 功能要求
1. 解析教师转录模板，识别【问题】-【停顿】-【答案】的固定模式
2. 在学生 ASR 转写结果中定位对应的"问题"和"答案"文本
3. 提取位于"问题"和"答案"之间的学生回答内容
4. 对于没有实质性回答的情况，标记为"未作答"
5. 输出符合指定 JSON 格式的标注结果

## 技术实现

### 架构设计
- 基于 `scripts/gemini_client.py` 的 Gemini LLM 客户端架构
- 复用现有的错误处理、重试机制和 JSON 解析逻辑
- 集成 `prompts/annotation.txt` 作为系统提示词

### 数据处理流程
1. **数据加载**: 读取教师转录文本、题库 JSON 和学生 ASR JSON
2. **内容提取**: 从 ASR JSON 中提取学生回答的文本内容
3. **结构化处理**: 使用 Gemini LLM 按照提示词要求处理数据
4. **结果输出**: 生成 `4_llm_annotation.json` 标注文件

### JSON 输出格式
```json
[
  {
    "card_index": 1,
    "问题": "simple",
    "学生回答": "简单的",
    "答案": "简单的，简易的；朴素的，简朴的"
  },
  {
    "card_index": 19,
    "问题": "rise",
    "学生回答": "未作答",
    "答案": "（太阳，月亮等）上升；上涨；起床，起立"
  }
]
```

## 约束条件

- 必须保持与现有代码架构的兼容性
- 不得修改现有的 LLM 评测逻辑
- 输出格式必须严格符合 JSON 规范
- 需要处理各种边界情况（无回答、识别错误等）

## 验收标准

1. 能够正确处理 Zoe51530-9.8 数据集中的 Phoebe 学生数据
2. 输出的 JSON 格式正确，包含所有必需字段
3. 能够准确识别学生回答和"未作答"状态
4. 处理时间在合理范围内（单个学生文件 < 30 秒）
