2. 由于`学生音频转录文本`包含老师声音和学生回答，所以属于`answer`答案字段的通常会出现不止一次，比如：mouse。老鼠。老鼠。elephant。大象。大象。tiger。老虎。老虎。lion。狮子。狮子。wolf。狼。狼。panda。熊猫。熊猫

严格按照以下步骤进行判断：

## 步骤1：判断题库的 question 语言

**重要**：只看 question 字段是中文还是英文！

从题库的前 3-5 个条目判断:
- 如果 `question` = 中文词 → 题库question语言是**中文**
- 如果 `question` = 英文词 → 题库question语言是**英文**

## 步骤2：提取 ASR 实际模式（综合分析）

**重要**：不要只看第一个词对，要综合分析多个词对！

从 ASR 转写的**全部内容**（或至少前 500 字符）中:

1. 找出**所有清晰的词对**（如"小孩 kid"、"不 not"、"kid 小孩"）
2. 统计每个词对的模式:
   - 中文在前，英文在后 → 记为 **中→英** (+1)
   - 英文在前，中文在后 → 记为 **英→中** (+1)
3. 计算主导模式:
   - 如果 中→英 数量 > 英→中 数量 × 1.5 → ASR 模式是 **中→英**
   - 如果 英→中 数量 > 中→英 数量 × 1.5 → ASR 模式是 **英→中**

## 步骤4：检查其他异常

**词汇匹配度**:
- 题库中的词汇在转写中匹配度 < 50% → `FAIL` - `WRONG_QUESTIONBANK`

**音频完整性**:
- 转写过短（< 题库数×5 字符）→ `FAIL` - `AUDIO_ANOMALY`
- 只有零散词语，无句子结构 → `FAIL` - `AUDIO_ANOMALY`

## 步骤5：输出判定结果

只输出 JSON，不要额外解释。

# 输出要求

- 必须是严格的 JSON 格式
- 只包含 `status` 和 `issue_type` 两个字段
- 键和字符串值都必须使用双引号 `""`
