## 4. Token 使用分析

### 输入 Token 组成
- 音频文件（转换为 token）
- ASR 转录文本
- 题库内容（Question Bank JSON）
- 系统提示词（System Instruction）
- 用户提示词模板（User Prompt Template）

### 输出 Token 组成
- JSON 格式的批注结果
- 包含：评分等级、错误计数、详细批注列表

### 观察
1. **输入输出比例**: 约 3.5:1（输入 token 是输出 token 的 3.5 倍）
2. **输出 token 稳定性**: 输出 token 非常稳定（1350-1381），说明批注格式一致
3. **输入 token 差异**: 输入 token 有一定波动（4411-5043），可能因为：
   - 音频长度不同
   - ASR 转录文本长度不同
4. **总 token 差异**: Lucy 的总 token 最高（14,933），可能使用了缓存机制

## 5. 成本优化建议

### 当前情况
- 单学生成本约 **2.6 美分**
- 处理 100 学生约需 **$2.60**
- 处理 1000 学生约需 **$26.00**

### 优化方向
1. **启用 Context Caching**：可减少重复输入内容的成本（题库、提示词模板）
2. **批量处理**：利用缓存机制处理同批次学生
3. **提示词优化**：精简 system instruction 和 prompt template（在不影响质量前提下）
4. **音频预处理**：压缩音频文件可能减少 token 使用
