## 5. 思考 Token 分析

### 观察
1. **思考 token 波动大**: 从 3,233 到 8,897，差距 2.75 倍
2. **无明显规律**: 与输入长度、音频长度无明显相关性
   - Allen: 思考最多 (8,897)，输入中等 (4,884)
   - Apollo: 思考最少 (3,233)，输入最多 (5,043)
3. **可能影响因素**:
   - 学生错误越多 → 思考越多？
   - 批注复杂度越高 → 思考越多？
   - 模型内部推理路径

### 成本影响
- **思考 token 成本占比**: 89.9% 的总成本来自输出（大部分是思考）
- **优化难度**: 思考过程由模型自动决定，难以直接控制

## 6. 成本优化建议

### 短期优化（有限效果）
1. **Context Caching**: 缓存题库、提示词模板
   - 预计节省输入成本 ~40%
   - 总成本节省 ~4%（因为输入占比小）

2. **精简提示词**: 减少 system instruction 和 prompt
   - 预计节省输入 token ~20%
   - 总成本节省 ~2%

3. **音频压缩**: 降低音频质量/采样率
   - 预计节省音频 token ~30%
   - 总成本节省 ~4%

### 长期优化（需要实验）
1. **禁用思考模式**（如果可能）
   - 查看是否有配置参数禁用 extended thinking
   - 可能影响质量，需要 A/B 测试

2. **切换模型**:
   - 尝试 `gemini-2.0-flash-thinking-exp` (可能更便宜)
   - 或普通 `gemini-2.0-flash` (无思考模式)

3. **混合策略**:
   - 简单案例用 Flash（快速+便宜）
   - 复杂案例用 Pro + Thinking

### 最大优化潜力
- 如果完全消除思考 token: **-81%** 成本（从 $0.097 降至 $0.018/学生）
- 如果只优化输入: **-4%** 成本（效果有限）
