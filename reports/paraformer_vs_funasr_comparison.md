## 2. 关键差异分析

### 2.1 句子分段策略

**Paraformer-v2**: 倾向于**长句合并**，单个句子可能跨越 10-40 秒

```
[1] 260ms - 10880ms (10.6秒)
    "Simple词，简单的简单的简易的、朴素的、简朴的。"

[2] 10980ms - 39060ms (28秒)
    "Complete形容词完成的完全的完整的完全的、彻底的complete及物动词完成完成list名词目录清单目录一览表。"
```

**Fun-ASR**: 倾向于**短句细分**，按语义/停顿拆分更细

```
[1] 880ms - 3360ms (2.5秒)
    "Simple 形容词，简单的。"

[2] 5140ms - 5340ms (0.2秒)
    "嗯。"

[3] 6910ms - 9470ms (2.6秒)
    "简单的、简易的、朴素的、简朴的。"

[4] 11500ms - 13260ms (1.8秒)
    "complete形容词。"
```

### 2.2 词级别时间戳精度

**Paraformer-v2** - 字符级拆分（中文按单字）:
```
     260ms -   2240ms : Simple
    2240ms -   3230ms : 词
    3230ms -   4220ms : 简
    4220ms -   5210ms : 单
    5210ms -   6200ms : 的
```

**Fun-ASR** - 词组级拆分（更接近语义单元）:
```
     880ms -   1200ms : Simple
    1200ms -   1600ms :  形
    1600ms -   1880ms : 容
    1880ms -   1960ms : 词
    2280ms -   3360ms : 简单的
```

### 2.3 转写文本差异

| 内容 | Paraformer-v2 | Fun-ASR |
|------|---------------|---------|
| 英文单词 | `Simple词` | `Simple 形容词` |
| 词性标注 | 部分丢失 | 更完整 |
| 停顿词 | 较少 | 保留"嗯"等 |
| 标点 | 顿号为主 | 逗号+顿号混合 |
| 英文格式 | 无空格 | 有空格分隔 |

## 4. 结论与建议

| 场景 | 推荐模型 | 原因 |
|------|----------|------|
| 需要细粒度时间对齐 | **Paraformer-v2** | 字符级时间戳更精确 |
| 需要语义分段 | **Fun-ASR** | 句子划分更符合自然停顿 |
| 英文+中文混合 | **Fun-ASR** | 英文识别格式更好（有空格） |
| 长音频批处理 | **Paraformer-v2** | 句子数少，处理效率高 |
| 口语评测应用 | **Fun-ASR** | 保留更多语气词和停顿 |
